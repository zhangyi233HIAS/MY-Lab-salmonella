# MY-Lab-salmonella

## 基于全基因组测序的多细菌病原爆发案例的耐药演化智能分析与精准用药决策系统

---

## 模块B：耐药表型预测与精准用药决策

### B.1 研究内容

#### 输入（来自模块A的输出）：

- 216株的AMR基因存在/缺失矩阵（已确认）
- 各菌株的MLST/CC分型（已确认）
- 优势克隆列表及其基因载荷（已确认）
- CARD数据库RGI输出结果

#### 输出：

- 每株菌对各类抗菌药的预测MIC值
- S/I/R判定矩阵
- 针对优势克隆的用药推荐报告

### B.2 技术路线（分三步递进）

#### Step 1：规则基线构建（CARD驱动）

```
CARD RGI输出 → AMR基因 → 匹配耐药机制分类
 ├── β-内酰胺酶（ESBL/AmpC/碳青霉烯酶）
 ├── 氟喹诺酮靶点突变（gyrA/parC）
 ├── 氨基糖苷修饰酶
 ├── 四环素外排泵/核糖体保护蛋白
 ├── 磺胺耐药（sul1/sul2/dfrA）
 └── 多黏菌素（mcr-1~mcr-10）
```

**规则逻辑：**

- 存在某耐药基因 → 查CARD机制分类 → 映射到具体药物 → 赋予预测MIC
- 多基因叠加时取最高MIC
- 记录置信度标签：High（直接耐药机制）/ Medium（间接关联）/ Low（推测性）

#### Step 2：MIC预测模型构建

**没有实测MIC时，模型策略：**

| 策略 | 数据来源 | 方法 | 可行性 |
|------|--------|------|--------|
| 策略A（首选） | CARD curated phenotype数据 | 基因组合 → MIC分布映射 | ✅ 可立即执行 |
| 策略B | 文献挖掘 | 同菌种同ST的MIC文献报道统计 | ✅ 补充验证 |
| 策略C | 外部公开数据集 | NCBI AMRFinderPlus关联的PATRIC表型数据 | ✅ 扩大训练集 |

**模型架构：**

```
输入特征层：
├── AMR基因特征（二值向量，~200维）
├── 突变特征（QRDR区域gyrA/parC突变，如有）
├── 质粒特征（携带的replicon类型，二值向量）
├── 克隆分型特征（ST/CC embedding）
└── 物种特征（one-hot）

 ↓

特征工程层：
├── 基因共现模式（关联规则挖掘：Apriori/FP-Growth）
├── 基因-质粒关联特征
└── 克隆-耐药载荷复合特征

 ↓

模型层（多药物多任务）：
├── 基础模型：梯度提升树（XGBoost/LightGBM）
│ └── 每种药物一个回归器，输出log2(MIC)
├── 增强模型：随机森林集成
│ └── 与XGBoost取均值，降低方差
└── 断点映射：EUCAST/CLSI生态位 → log2(MIC) → S/I/R

 ↓

输出层：
├── 预测MIC值（μg/mL，每种药物）
├── S/I/R判定（附置信度）
└── 不确定性区间（模型ensemble的标准差）
```

#### Step 3：用药推荐引擎

```
输入：优势克隆的预测耐药谱
 ↓
约束过滤：
├── 兽药目录合规性（中国兽药典2025版）
├── 人畜共用药物标记（氟喹诺酮类慎用）
├── 给药途径约束（口服/注射/饮水）
└── 联合用药禁忌检查
 ↓
评分排序：
├── 敏感性得分 = Σ(S判定药物权重) / Σ(检测药物数)
├── 安全性扣分 = 人畜共用药物 × 0.7
├── 广谱性加分 = 覆盖Top3优势克隆的药物 +0.2
└── 最终得分 = 敏感性 × 安全性 × 广谱性
 ↓
输出：推荐用药方案
├── 一线方案（得分最高）
│ ├── 药物名称 + 剂量参考
│ ├── 覆盖克隆列表
│ ├── 预期敏感性
│ └── 注意事项
├── 备选方案（得分次高）
└── 耐药预警（如果一线方案存在潜在选择压力风险，关联模块C）
```

### B.3 验证策略（关键评审点）

**验证层级1 — 内部一致性：**
- AMR基因组合是否与已知耐药机制逻辑一致？
- 例：同时携带bla_CTX-M + bla_TEM → β-内酰胺类MIC应升高

**验证层级2 — 跨源比对：**
- 将预测结果与以下来源比对：
  - CARD数据库中的curated phenotype记录
  - CHINET中国细菌耐药监测网（同菌种耐药率对比）
  - 已发表的鸽源/禽源病原菌耐药文献

**验证层级3 — 交叉验证（如有外部数据）：**
- 从PATRIC/NCBI获取同菌种ST型的实测MIC数据
- 模型预测 vs 实测 → 计算分类准确率（S/I/R）

**如果均无法获得实测数据：**
- 明确标注为"规则推断 + 机器学习预测"
- 在申请中诚实说明局限性
- 提出后期补充实测药敏的计划作为验证闭环

## 模块C：给药后优势克隆种群演化预测

### C.1 研究内容

**核心问题：** 对当前爆发的优势克隆施加某种药物选择压力后，种群结构会如何变化？哪些克隆会被抑制？哪些会扩张？

#### 输入（来自模块A+B）：

- 当前克隆种群分布（频率 + 基因载荷）
- 各克隆对各类药物的预测MIC/SIR
- 质粒传播风险评分（模块4a-4d）

#### 输出：

- 不同用药方案下的克隆种群动态曲线（72h / 7d / 30d）
- 最优方案推荐（抑制耐药克隆 + 保护敏感菌群）

### C.2 模型架构

```
┌─────────────────────────────────────────────┐
│ Agent-Based 种群动力学模型                  │
├─────────────────────────────────────────────┤
│                                             │
│ 每个克隆CC/ST = 一个Agent                   │
│ Agent状态：                                  │
│ ├── N(t): 当前种群规模（相对丰度）          │
│ ├── R(t): 耐药表型向量 [对各药物的S/I/R]   │
│ ├── P(t): 携带的质粒集合                    │
│ ├── V(t): 毒力载荷                          │
│ └── fitness(t): 适应度                      │
│                                             │
│ 状态转移规则：                               │
│ ├── 无药状态：                              │
│ │ dN/dt = r × N × (1 - N/K)               │
│ │ r = 基础生长速率（物种特异性）            │
│ │ K = 环境容纳量                            │
│ │                                          │
│ ├── 有药状态（药物i施加选择压力）：         │
│ │ if S药物i: r_eff = r × (1 - E_i)        │
│ │ if I药物i: r_eff = r × (1 - E_i/2)      │
│ │ if R药物i: r_eff = r × (1 + C_i)        │
│ │                                          │
│ │ E_i = 药物i的杀菌效率                     │
│ │ C_i = 耐药克隆在药物i压力下的             │
│ │       竞争优势补偿系数                    │
│ │                                          │
│ ├── HGT事件（每步以概率p_hgt触发）：       │
│ │ 耐药质粒从耐药克隆 → 敏感克隆            │
│ │ p_hgt = f(共培养时间, 质粒相容性,        │
│ │          接合转移频率文献值)              │
│ │ 转移后果：敏感克隆R(t)更新，             │
│ │          可能转变为中介/耐药              │
│ │                                          │
│ └── 适应度代价：                            │
│   耐药基因携带 → fitness惩罚                │
│   f_cost = 1 - Σ(各基因的fitness cost)    │
│   → 无药环境下耐药克隆可能被淘汰            │
│                                             │
├─────────────────────────────────────────────┤
│ 模拟场景：                                   │
│ ├── Scenario 0: 不给药（基线对照）         │
│ ├── Scenario 1: 方案A单药治疗               │
│ ├── Scenario 2: 方案B单药治疗               │
│ ├── Scenario 3: 方案A+B联合用药             │
│ └── Scenario 4: 轮换用药（A→B→A...）      │
└─────────────────────────────────────────────┘
```

### C.3 参数校准方案

| 参数 | 校准来源 |
|------|--------|
| 基础生长速率 r | 文献值（E. coli/K. pneumoniae体外生长曲线） |
| 药物杀菌效率 E_i | 文献值（各药物的杀菌曲线PK/PD参数） |
| 耐药补偿系数 C_i | 推算：C_i = MIC_R / MIC_S × α（α=0.1~0.3） |
| HGT概率 p_hgt | 文献值（IncF/IncI等质粒的接合频率10^-2~10^-6） |
| fitness cost | 文献值（各AMR基因的适应度代价0~0.15） |
| 环境容纳量 K | 标准化为1（相对丰度） |

**校准验证：**
- 用基线场景（Scenario 0）验证模型能复现当前种群分布
- 敏感性分析：各参数±50%时，结论是否稳健
- 最坏/最好情况边界：参数取极端值时的预测范围

### C.4 预测输出（申请书写法）

对每个用药方案，输出：

#### 1. 克隆种群动态图
- X轴：时间，Y轴：各克隆相对丰度堆叠面积图
- 72h：急性期效果（药物是否快速抑制目标克隆）
- 7d：中期趋势（是否出现耐药克隆反弹）
- 30d：长期风险（是否诱发新的耐药优势克隆）

#### 2. 耐药传播风险评分
- HGT事件累计次数
- 新产生耐药克隆的概率
- 质粒介导的跨克隆耐药扩散趋势

#### 3. 方案综合评分

| 方案 | 抑菌效果 | 耐药选择压力 | HGT风险 | 综合推荐 |
|------|--------|-----------|--------|---------|
| 方案A | ★★★★ | ★★★☆ | ★★☆☆ | 推荐 |
| 方案B | ★★★☆ | ★★☆☆ | ★★☆☆ | 备选 |
| A+B联合 | ★★★★★ | ★★☆☆ | ★★☆☆ | 首选 |
| 轮换用药 | ★★★☆ | ★☆☆☆ | ★★★☆ | 慎用 |

#### 4. 关键发现与建议

示例输出：
- "方案A+B联合用药在7天内可将目标克隆丰度降低至<5%"
- "但30天后观察到mcr-1质粒通过HGT扩散至CC10克隆的风险为中等"
- "建议联合用药疗程不超过14天，并加强用药后监测"

## 项目总体架构

该系统整合了基因组学、耐药表型预测和群体动力学模型，为多细菌病原爆发的精准用药提供数据驱动的决策支持。

**关键特点：**
- ✅ 基于实际测序数据的AMR基因组合分析
- ✅ 机器学习驱动的MIC预测
- ✅ 质粒传播风险评估
- ✅ 用药方案的群体动力学模拟
- ✅ 透明的验证策略和误差评估
